Asincronia HTML

  • Procesamiento Single Thread y Multi Thread

    • JavaScript tiene funciones que nos va a permitir lanzar acciones despues de cierto tiempo o una n cantidad de veces

    JavaScript es un lenguaje Single Thread. No nos tenemos que preocupar por los problemas de concurrencia, recolector de basura, etc, eso lo hace el lenguaje de programación. No se puede hacer operaciones largas, como el acceso a la red, porque se bloquea el hilo principal.

    La sincronia permite hacer largas solicitudes de red sin bloquear el hilo principal. JS trabaja con un modelo asincrono y no bloqueante y tiene un loop de eventos de un solo hilo para operaciones de entrada y salida.

    Los hilos son las unidades basicas de ejecucion de cada proceso que realiza nuestra maquina. En nuestra computadora se levanta un proceso, internamente esos procesos pueden correr varios hilos o un solo hilo. Dependiendo del lenguaje, seran single thread o multi thread

  • Operaciones de CPU y Operaciones I / O

    • En un procesamiento, en el codigo de una aplicacion podemos tener operaciones de este tipo. Las operaciones que se consideran de CPU son aquellas que se consideran que van a pasar mayor tiempo consumiendo operaciones de CPU, por ejemplo: Un loop infinito consumira más y más memoria y, de esta manera, consumira más rendimiento de la maquina.

    Las Operaciones de Input / Output seran aquellas que estaran esperando la mayor parte del tiempo la peticion del recurso que han solicitado.

  • Operaciones concurrentes y paralelas

    • La concurrencia es cuando dos o más tareas progresan simultaneamente, se ejecutan al mismo tiempo y van avanzando simultaneamente, el paralelismo es cuando dos o más tareas trabajan al mismo tiempo. La clave esta en la palabra progreso.

  • Operacion bloqueante y no bloqueante

    • Se refiere a la fase de espera cada que se ejecuta una operacion en nuestro codigo, se refiere a como toma esa fase de espera. Una bloqueante no devuelve el control a la aplicacion hasta que termina de realizar toda la tarea. Una no bloqueante se ejecuta y devuelve el control así no termine la tarea que este realizando.

  • Operaciones sincronas y asincronas

    • Se refiere a cuando obtendra respuesta. Sincrono significa que la respuesta sucede en el presente, o sea, en el tiempo inmediato. Asincrona significa que la respuesta sucede en el futuro, se ejecuta pero no va a saber cuando va a venir la respuesta, no espera el resultado.
